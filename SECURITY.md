# Security Policy

Purpose

This document describes how security is handled in this project.

Reporting Vulnerabilities

If you discover a security issue:

Do not open a public issue

Contact the maintainers privately

Provide steps to reproduce and impact

We will acknowledge and investigate promptly.

Security Principles

Least privilege access

Strict tenant isolation

No secrets in code or logs

Rate limiting and abuse protection

Dependency updates and patching

Sensitive Data

Never commit:

API keys

tokens

credentials

production data

Incident Handling

Security incidents follow the same process as operational incidents but with restricted visibility.

See: docs/runbooks/incident.md

End of Security Policy
